Project Purpose & Rules

The goal is to demonstrate semantic HTML skills with minimal CSS and no JavaScript[cite: 27, 28].

  • Strict CSS: Only color, background-color, font-family, and font-size are allowed[cite: 52, 167].
  • No Layout: Properties like margin, padding, or flex are strictly forbidden.
  • Content: Must only use material from Chapters 9 and 12 of Dix et al[cite: 33].

HTML Requirements

Semantic HTML and Accessibility Rules
Area Rule
Structure Use <header>, <nav>, <main>, and <footer>[cite: 36, 144].
Navigation Must be an unordered list (<ul>) inside a <nav>[cite: 38].
Accessibility Use aria-current="page" on the active link and alt text on all images[cite: 49, 166].

Lab Exam Practice

Part A: Fix the Mistakes

Identify common errors like skipping heading levels (e.g., <h1> to <h3>) or missing <li> tags in menus[cite: 42, 57, 175].

Part B: True/False

  • T/F: KLM predicts execution time for expert performance. (True) [cite: 79, 187]
  • T/F: A page can have multiple <h1> tags. (False) [cite: 40, 151]
