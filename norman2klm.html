The Keystroke-Level Model (KLM)

KLM is a low-level instance of GOMS. While Norman focuses on why a user does something, KLM predicts how long it takes for an expert user to execute it.

The Seven Primitive Operators

  • K (Keystroke): Pressing a key on the keyboard.
  • B (Button): Pressing or releasing a mouse button.
  • P (Pointing): Moving the mouse to a target on the screen.
  • H (Homing): Moving hands between keyboard and mouse.
  • D (Drawing): Drawing straight line segments.
  • M (Mental): Mental preparation or decision making.
  • R (Response): System response time (waiting for the computer).

Task Analysis: Correcting a Payer Name

Building on the "Who Paid" problem, imagine an expert user realizes they typed "Alce" instead of "Alice" and must correct it:

  1. Move hand to mouse (H)
  2. Position cursor after the 'l' (P)
  3. Click to set insertion point (B)
  4. Return hand to keyboard (H)
  5. Mental pause to confirm position (M)
  6. Type the missing 'i' (K)

Predicted Operator Sequence: H → P → B → H → M → K

Critical Summary

Why does KLM only have 7 operators?
Unlike Norman’s Action Cycle, KLM does not model the Gulf of Evaluation. It assumes the user already knows the interface perfectly (expert) and makes no mistakes. Therefore, it ignores the time spent perceiving or interpreting the screen, focusing strictly on physical execution and necessary mental pauses.
